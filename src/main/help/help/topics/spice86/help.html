Spice86 Runtime Import And Generation

The Spice86 plugin restores the original Spice86 workflow inside Ghidra: import runtime evidence from a dump directory, use it to improve Ghidra's view of the DOS program, and then generate Spice86 C# overrides from the resulting model.

Menu Actions

  • Spice86 -> Import runtime evidence
  • Spice86 -> Generate C#

Imported Data

  • Recorded function names from spice86dumpGhidraSymbols.txt
  • Dynamic call and jump targets from spice86dumpExecutionFlow.json
  • Observed return targets
  • Bookmarks for executed code blocks
  • Bookmarks for executable byte writes when the dump contains them

How To Use

  1. Set SPICE86_DUMPS_FOLDER to the directory that contains the dump files.
  2. Open the matching program in Ghidra.
  3. Run Spice86 -> Import runtime evidence.
  4. Run Spice86 -> Generate C# once the program has been annotated.

The importer infers the linear delta between Spice86 runtime addresses and the current Ghidra program entrypoint, so it can work both with runtime-aligned images and standard DOS EXE imports.

Design Notes

This restore intentionally fixes the address-model mismatch between Spice86 runtime addresses and Ghidra program addresses, but it does not add a new memory-dump repair phase. The import side remains conservative: it does not delete existing references wholesale, does not repeatedly rebuild functions, and only creates new disassembly or functions at observed entrypoints when that is safe.
